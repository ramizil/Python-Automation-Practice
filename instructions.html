Playwright Automation Practice

Build UI + API test frameworks from scratch — the same way in Python · pytest TypeScript · @playwright/test

What you'll build

Two mirrored frameworks that test a frontend UI and a backend API (including mocks), against free public demo apps. Each topic has a worked reference example plus a practice exercise (skeleton with TODOs + a hidden solution). Do every topic in both languages to compare the idioms.

Clean REST API (auth + CRUD)restful-bookerrestful-booker.herokuapp.com

Dummy bank (capstone)Parabankparabank.parasoft.com

Setup (Windows / macOS / Linux)

Needs Git, Python 3.10+, Node.js 18+. Full details in SETUP.md. One command from the repo root sets up both stacks:

./setup.ps1        # Windows (PowerShell)
bash setup.sh      # macOS / Linux

PYTHON

cd python-playwright
python -m venv .venv
source .venv/bin/activate   # Win: .\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
playwright install
pytest                      # 14 reference tests

TYPESCRIPT

cd ts-playwright
npm install
npx playwright install
npx playwright test         # 14 reference tests
Corporate / proxy networks: if you hit "self-signed certificate in chain", it's already handled — IGNORE_HTTPS_ERRORS defaults ON. Set IGNORE_HTTPS_ERRORS=false to enforce real certificate checks.

How to work an exercise

  1. Read the topic's worked example in tests/ (run it first).
  2. Open exercises/NN_topic/EXERCISE.md and implement the skeleton.
  3. Run just that exercise (commands below) until it's green.
  4. Compare against the reference solution/.
  5. Repeat the topic in the other language.
